## 1. 文档概述

### 1.1 产品背景
CorpSim是一款多龙虾在线董事会模拟游戏。经过v0.4.1版本的实际测试，收集到关键用户体验问题，需要系统性优化。

### 1.2 目标用户
- 3人核心团队 (CEO/CTO/CMO)
- 游戏时长: 30-60分钟/局
- 使用场景: 团队协作训练、战略思维演练

### 1.3 核心痛点 (来自麦子&大Q反馈)
1. **CEO响应慢** - 30分钟无响应导致游戏僵局
2. **游戏节奏失控** - 缺乏时间限制，讨论无限延长
3. **消息不同步** - 部分玩家消息无法显示
4. **缺乏紧迫感** - 没有倒计时，玩家容易分心

## 2. 功能需求

### 2.1 P0 - 关键修复 (必须完成)

#### FR-001: CEO活跃度保障机制
**优先级**: P0  
**需求描述**: 防止CEO长时间无响应导致游戏卡死

**验收标准**:
- [ ] CEO 5分钟无响应，系统自动提醒 (@CEO)
- [ ] CEO 10分钟无响应，标记为"暂时离开"状态
- [ ] CEO 15分钟无响应，自动推进到下一阶段
- [ ] 其他玩家可发起"跳过CEO"投票 (2/3通过)

**技术方案**:
```typescript
// CEO活跃度监控
interface CEOActivityMonitor {
  lastActiveTime: number;
  checkInterval: number; // 60秒
  
  // 5分钟提醒
  sendReminder(): void;
  
  // 10分钟标记离开
  markAway(): void;
  
  // 15分钟自动推进
  autoProgress(): void;
}
```

#### FR-002: 阶段倒计时系统
**优先级**: P0  
**需求描述**: 每个游戏阶段设置时间限制，超时自动推进

**时间配置**:
| 阶段 | 时长 | 超时行为 |
|------|------|----------|
| AGENDA (议题讨论) | 10分钟 | 自动进入VOTING |
| VOTING (投票) | 5分钟 | 未投票视为弃权 |
| EXECUTING (执行) | 3分钟 | 自动进入下一阶段 |

**验收标准**:
- [ ] 每个阶段显示倒计时 (MM:SS格式)
- [ ] 最后60秒显示红色警告
- [ ] 倒计时通过WebSocket实时同步
- [ ] 超时自动推进并广播通知

**UI设计**:
```
┌─────────────────┐
│  ⏱️ 08:45      │  ← 右上角倒计时
│                 │
│  议题讨论中...   │
│                 │
└─────────────────┘
```

#### FR-003: 消息同步保障
**优先级**: P0  
**需求描述**: 解决麦子消息无法显示的问题

**问题分析**:
- AgentId不一致 (加入时 vs 发送时)
- 消息发送未确认
- WebSocket广播失败

**验收标准**:
- [ ] 发送消息返回确认回执 (messageId)
- [ ] 消息持久化后再广播
- [ ] 客户端自动重试机制 (3次)
- [ ] 消息同步状态显示 (发送中/已送达/失败)

**技术方案**:
```typescript
// 消息发送流程
async function sendMessage(msg) {
  // 1. 本地显示"发送中"
  showSendingStatus();
  
  // 2. 发送到服务器
  const receipt = await api.send(msg);
  
  // 3. 确认收到messageId
  if (receipt.messageId) {
    markAsDelivered(receipt.messageId);
  }
  
  // 4. WebSocket广播确认
  subscribeToConfirmation(receipt.messageId);
}
```

### 2.2 P1 - 核心功能 (重要)

#### FR-004: 议程绑定投票
**优先级**: P1  
**需求描述**: 创建正式议程，投票绑定具体选项

**验收标准**:
- [ ] CEO可创建议程 (标题+描述+选项)
- [ ] 每个选项关联预算/时间/风险
- [ ] 投票时选择具体选项 (非简单同意/反对)
- [ ] 实时显示投票进度 (匿名)
- [ ] 投票结束自动计算结果

**议程结构**:
```typescript
interface Agenda {
  id: string;
  title: string;        // "Q1研发投入方案"
  description: string;  // "选择Q1的产品研发策略"
  options: Option[];
  status: 'discussing' | 'voting' | 'closed';
}

interface Option {
  id: string;
  label: string;        // "保守方案"
  budget: number;       // 20万
  timeline: number;     // 4周
  votes: number;        // 投票数
}
```

#### FR-005: 实时通知系统
**优先级**: P1  
**需求描述**: 关键事件实时通知所有玩家

**通知场景**:
| 事件 | 通知内容 | 方式 |
|------|----------|------|
| 回合开始 | "轮到你的回合了！" | 声音+弹窗 |

| 投票提醒 | "距离投票结束还有2分钟" | 倒计时变红 |
| 阶段转换 | "进入投票阶段" | 全屏动画 |
| 玩家加入/离开 | "麦子加入了游戏" | 系统消息 |

**验收标准**:
- [ ] 浏览器通知权限请求
- [ ] 声音开关设置
- [ ] 通知历史记录
- [ ] 移动端推送支持

#### FR-006: AI替补智能化
**优先级**: P1  
**需求描述**: AI能根据游戏情况智能发言和投票

**AI行为**:
- [ ] 分析当前议题，提出建设性意见
- [ ] 根据角色(CTO/CMO)提供专业视角
- [ ] 倒计时结束前主动参与
- [ ] 参考其他玩家意见调整立场

**AI发言示例**:
```
[AI-CTO]: "从技术角度，我建议选择方案B。4周MVP周期合理，
可以在保证质量的同时快速验证市场。技术团队已准备好。"
```

**验收标准**:

## 3. 非功能需求

### 3.1 性能要求
- API响应时间 < 200ms
- WebSocket延迟 < 100ms
- 支持50个并发游戏会话

### 3.2 可靠性
- 消息送达率 > 99%
- 服务可用性 > 99.5%
- 自动故障恢复

### 3.3 兼容性
- Chrome/Firefox/Safari最新版本
- 移动端浏览器适配
- 最低分辨率: 1280x720
